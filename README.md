## What I Build

## Projects (My Growth in Public)

### TideScore — *Rule-Based Credit System*

An alternative credit scoring system for Nigerians without formal credit history.

Banks rely on records many people don’t have.
TideScore explores different signals — behavior, patterns, proxies.

It’s not perfect. But it works, and it’s honest about its limits.

**What I learned:**

* Translating real-world behavior into logic
* Working with imperfect data
* Building scoring systems from scratch

---

### GoalBox — *Accountability System*

A personal commitment tracker.

Not a habit app. Not a to-do list.
Something in between — designed to make you feel the weight of what you said you’d do.

Started as something I needed. Then became something others could use.

**What I learned:**

* Behavioral design
* Simplicity over feature bloat
* Turning personal tools into usable products

---

### TrustCheck — *Trust & Relationship System*

My most complete system so far.

Built around a simple reality:

> In many places, especially here, transactions run on trust — not formal systems.

TrustCheck models that:

* tracks relationships
* records interactions
* exposes patterns of reliability

It includes:

* a backend API
* a working frontend
* actual logic that reflects real-world use

**What I learned:**

* API design (FastAPI)
* structuring systems end-to-end
* thinking beyond features → into systems

## How I Work

* **I build before I fully understand**
  Waiting to “know enough” slows everything down. Building teaches faster.

* **I ship imperfect things**
  A working system with rough edges beats a perfect idea that never leaves my head.

* **I care about the real problem**
  Not theoretical problems. Not abstract ideas. Real things people deal with.

* **I work within constraints**
  Limited resources, inconsistent infrastructure, incomplete data — that’s the environment. So I build for it.
